# Concept2Paper MVP

Сервис для поиска перспективных научных пар концептов на основе графа соавторства
и обученной модели link-prediction (MLP).

## Архитектура

```
┌──────────┐   HTTP   ┌────────────┐   Celery   ┌────────────────────┐
│ Streamlit│ ───────▶ │  FastAPI   │ ──────────▶ │ Worker (ML+LLM)    │
│  (UI)    │          │  REST API  │             │ × N (масштаб.)     │
└──────────┘          └────────────┘             └────────────────────┘
                             │                          │
                        ┌────▼──────┐          ┌───────▼──────┐
                        │ PostgreSQL│          │  Redis       │
                        │ (results) │          │  (broker)    │
                        └───────────┘          └──────────────┘
```

### Сервисы

| Сервис | Роль |
|--------|------|
| `postgres` | Хранение результатов сессий и черновиков |
| `redis` | Брокер задач Celery |
| `api` | REST API (FastAPI, порт 8000) |
| `worker` | Celery-воркер — ML-инференс + LLM |
| `ui` | Streamlit-интерфейс (порт 8501) |

Данные (граф, модель, эмбеддинги) хранятся как файлы — монтируются через Docker volume.

## Доменная модель

```
Session        — запрос пользователя (ключевое слово + статус)
  └─ ConceptPair — топ-5 пар концептов со скорами (хранятся в БД)

Draft          — LLM-черновик плана статьи для выбранной пары
```

## Данные и модель

Граф, эмбеддинги и веса модели лежат в `data/`. В PostgreSQL хранятся только результаты работы сервиса — сами артефакты ML остаются файлами.

| Файл | Описание |
|------|----------|
| `graph_2017_2024.pkl` | Граф концептов (47k узлов, 326k рёбер) |
| `article_embeddings.pkl` | Эмбеддинги статей (allenai-specter) |
| `mlp_all.pt` | Дообученная MLP (256→128→64→1) |

## LLM

Для генерации черновиков используется **Groq API** (модель `llama-3.1-8b-instant`) — бесплатный тир, регистрация на [console.groq.com](https://console.groq.com).

Ключ передаётся через переменную окружения `GROQ_API_KEY`. Без ключа сервис работает в полном объёме, но вместо AI-черновика возвращает шаблон.

## REST API

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/v1/concepts` | 20 ключевых концептов |
| `POST` | `/api/v1/sessions` | Создать сессию инференса |
| `GET` | `/api/v1/sessions/{id}` | Статус + результаты сессии |
| `POST` | `/api/v1/drafts` | Генерация черновика через LLM |
| `GET` | `/api/v1/drafts/{id}` | Получить черновик |

## Фичи модели

Для каждой пары концептов вычисляются 43 признака:

- **16 граф-топологических**: Jaccard, Adamic-Adar, частоты, тренды и др.
- **25 sparse-матричных**: степень узлов и AA-скор по 5 годам (2020–2024)
- **2 embedding**: cosine + L2 между усреднёнными эмбеддингами статей концептов

Модель: MLP (256→128→64→1) обученная с contrastive loss на графе 2017–2024.
